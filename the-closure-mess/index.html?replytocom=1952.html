The Closure Mess

Posted on June 8, 2005 by Kvanvig JonJune 8, 2005

I’ve been working on closure for awhile, and I’ve come to the conclusion that the literature is a mess about the idea. To a certain extent, this post is a plea for terminological regimentation, so it might be viewed as just a terminological issue. I’ll say why it isn’t just that near the end.

To see the confusions, let’s begin by supposing we have a battery of epistemic concepts. In addition to these concepts, we’ll also have a bunch of epistemic principles (some true, we suppose, some false). The principles will relate various conditions to the instantiation of some epistemic concept. So here’s some examples:
if you know, then you are justified in believing.
if you take yourself to be justified, you are justified.
if you’re appeared to in a certain way, you’re justified.
if you are in a position to know and believe the claim as well, then you know.

What makes these principles epistemic is that they deal with epistemic concepts.

Among the epistemic principles will be closure principles. The key question is the question of how to tell when an epistemic principle is legitimately termed a closure principle. Here’s the beginning of the story that will tell us how to regiment our talk about closure.

Closure is a logico-mathematical concept. Strictly speaking, it is a function on an object, and the right terminology says that an object is closed just when the function on the object is equal to the object.

This logico-mathematical background is the source of the search for closure principles in contemporary epistemology. Note what we can learn from this source. First, when asking about correct closure principles, we are asking about whether something is closed. If we want to be extremely picky, we should insist that we be able to identify some object and some function on an object before we’re willing to say that we’ve found a true closure principle.

But suppose we relax a little. Suppose we find an object and then perform some operation on that object and argue that, in some important way, what we end up with can be identified with what we started with. A little loosey-goosey, for sure, but it still has some bite. It still has some bite because the key idea is that we have to argue that something is closed, and to be closed, we have find what we began with and what we end up with as being equal or identified.

So now go back to our assortment of epistemic principles. An epistemic principle is a conditional with epistemic concepts in there somewhere. In order to be a closure principle, the epistemic principle must identify something that is closed. The bite of this requirement is that, to do so, the principle must cite the same epistemic concept in antecedent and consequent. That’s what makes the following a paradigm example of a closure principle:

If q follows from what you know, then you know q.

It also explains why the following is not a closure principle:

If a certain group of claims is justified and that group also meets some coherence requirement, then one is in a position to know that the members of that group are true.

The first principle cites the concept of knowledge in both antecedent and consequent. It also identifies some function or operation to perform on items that fall under the concept, and says that anything resulting from the application of that function or operation also falls under that concept. So what we are considering before the application of that function or operation can be identified with what we have after this application: we consider the class of things known, create a list of the deductive consequences of the class, and note that the union of this list with what we began is just the same as what we began with (if the principle is true, that is).

As we move away from this paradigm closure principle, things get muddier. But if we still insist that something is closed, we can throw out any epistemic principle that doesn’t cite the same epistemic concept in antecedent and consequent. So, for example, the following still pass scrutiny:

if you know that q follows from what you know, then you know q.

if you competently deduce q from what you know, then you know q.

Notice that this last principle doesn’t involve the application of a mathematical function to the set of things known, but I think we can live with this level of metaphorical extension. It says if you start with a body of knowledge and undergo a certain psychological process or operation, then what you end up with is knowledge. I think we can live with calling this a closure principle for knowledge, because it claims that something is closed, with an identification of a certain sort: you start with stuff you know and you end up with stuff you know.

Now, if some wish to be picky and insist that this isn’t really closure (because there is really no object that is equal to some function on that object), we will understand. It’s not really closure. But it bears a strong enough metaphorical connection to the real thing that it seems not worth arguing about. And the basis for this opinion is that, in some way, it’s still trying to specify something that is closed.

But now think of the array of epistemic principles that don’t at all try to specify that something is closed because they don’t even use the same epistemic concepts in both antecedent and consequent. Some say that if q follows from what you know, then you are in a position to know that q is true; others say that if you justifiably believe that a conditional is true while justifiably believing its antecedent, the consequent of the conditional is justified for you; still others say that if you know q follows from what you know, then you have all you need, evidentially speaking, to know that q is true.

Defenders of these principles call them closure principles, but I can’t see why. I can’t even find a metaphorical purchase on the idea that there is something here that is closed. So I say, these are epistemic principles, but they are not closure principles.

The alternative, as I see it, is to say that a closure principle in epistemology is any principle that says something about the relationship between deduction and some particular epistemic concept. More generally, we could also talk about inductive closure principles, probabilistic closure principles, and the like. The use of ‘closure’ in such a language is, as far as I can see, completely superfluous. If you’re not in some way trying to say that something is closed, just say that you’ve found an important epistemic principle. You might also say it has something to do with deduction or some other notion of implication. But don’t say it’s a closure principle.

There. I feel better…

    It seems to me that Jon’s formulation of closure here is a little off, by being framed in terms of objects (something like f(a) = a) rather than sets. I think that what we want is that a set A is closed on a function f (x,y) iff
    (x)(y)(z)[((f(x,y) = z) & (x is in A) & (y is in A)) –> z is in A]
    (of course f could be a function with as many arguments as you like, but it’s obvious how to change the formulation so as to generalize.)

    I don’t even think it’s required that the image of A on f be A itself (i.e., f(A) = A), since all that’s needed is that the image be a subset of the original set. E.g., the natural numbers are closed on the successor function, but the image of the successor function does not include 0. Indeed, one also often talks about taking the ‘closure’ of a set B under f, such that f(B) is the smallest set that has B as a subset and is also closed under f; B will often turn out to be a strict subset of f(B). (It is thus a necessary and sufficient condition for a set B’s being closed on f that B is identical to B’s closure on f — maybe that’s what Jon has in mind?)

    So, if we let A be the set of known things, and f(x,y) be a function that takes known things to that which is competently deduced from them (which probably isn’t strictly speaking a _function_, in mathematical terms, but I don’t think that’s what’s at issue), then we do get a closure principle that is well expressed by “if you competently deduce q from what you know, then you know q.”

    I’m not sure anything deep hinges on this, and in particular it is completely consistent with Jon’s basic point about having the same concepts in both antecedent and consequent. But I’m not 100% sure that nothing hinges on it, either.

  5. Gregory Wheeler on June 8, 2005 at 3:43 pm said:

    I agree with Jonathan: I’m not sure what it means for an object to be closed, but I do have a grip on a set being closed on a function or a class of functions. I suppose that what one might mean by ‘object’ is a structured set of some kind, but it is still some type of collection or list.

    One thing that changes when moving from talk about functions on sets to talk about functions on doxastic states–or some suitable carrier for the epistemic property you’re interested seeing preserved under the operations in question–is that the process by which the image set is constructed becomes salient. In the case of epistemic closure, you aren’t simply interested in the semantic question of whether the proposition that “q follows from what S knows” entails that “S knows q”, but something roughly analogous to the proof theory of getting from the LHS to the RHS in this example: i.e., one wants to specify the conditions under which it is epistemically permissible for an agent to move from the LHS to the RHS.

    Perhaps, then, some of the mess comes from whether one approaches this topic from a syntactic point of view or from a semantic point of view. If you have in mind how to construct a new set of items from known items, you’re likely to think of exceptions and epicycles and come up with a description like knowing on the LHS ‘puts you in a position’ to know on the RHS. If you start at this semantically, you might be less interested in this level of detail but, rather, wonder what holds given that all is constructed in the right manner. Upshot: maybe the complaint about the same concepts appearing on both sides of the relation is a consequence of this semantic view of the problem.

    Another thought (from the syntactic view): might we be better to talk about a closure algorithm rather than a closure principle? Two reasons in favor of this suggestion: one, epistemic closure is clearly operational in character and we have mathematics for verification and analysis of algorithms that is richer than simply functions on sets; two, there seems to be several components to even naive closure principles that would not only benefit from individual study (e.g., how justification propagation works; how belief change and revision works) but also would need to be understood in terms of how they interact when considering the question of when and under what conditions may we add new knowledge by faculties of reason.

  11. Gregory Wheeler on June 9, 2005 at 11:22 am said:

    Hi Jon,

    I think identity is interesting and far from trivial. The pre-theoretic idea behind identity is that it is the nothing function: apply it an argument, and you get that very same argument. Surprisingly enough, this can be tricky to formulate in expressive languages without running afoul of self-referential paradoxes. Witness, for instance, the difference between Church-style type theories and Curry-style type theories: within Church’s theory, there is no single term x.x; rather, there are as many distinct lambda-terms as there are distinct types. This is fine if you think of functions like a mathematician, but strange from a metaphysican’s point of view: surely identity isn’t simply a multitude of particular identity functions! It seems like the perfect candidate for a single-concept function! Curry-style type theories give you this, a single lambda-term applicable to a multitude of types, but only at a price: you can create invariances by manipulations (beta-conversions) within Curry-style systems that are otherwise identity preserving in Church-style theories.

    The point of this is that while our pre-theoretic notion of identity might seem well and fine, working with a formal notion of it (within second order logic, in this case) can be hairy and far from non-trivial. (Or, perhaps better put, working with highly expressive languagues that include identity is far from trivial.) In the end, a satisfactory account (of epistemic closure, say) will be constructed (as much as possible) from formal notions and, yes, highly expressive languages.

    So, for this reason, I would take such talk (in line 6) as ill-formed only because it tempts one to build upon an informal notion that provides for more treacherous footing than you’re lulled otherwise into believing.

  15. jon kvanvig on June 10, 2005 at 6:27 am said:

    I think neither (1′) nor (2′) is a closure principle. Here’s an intuitive account of mathematical closure, for sets. Take a set, and some operation or function on it–I’ll use the natural numbers and addition. The door to this set is closed under this operation: apply the operation to any of the members of the set, and the result is already in the set. The set is not closed when the door stands open, and other things can get in by application of the operation. For example, take the set of natural numbers and the operation of subtraction. That set is not closed.

    So, notice that closure involves, when dealing with sets, an original set and an image set, the latter being the set constructed from all possible applications of the operation or function. Closure obtains, or the set is closed, when these two sets are the same.

    Given that, I’ll say a couple of things about why (1′) isn’t a closure principle. It starts with, as you put it, a conjunction. Which conjunction? The conjunction of things you know. To be a closure principle, then, it should end with a conjunction of things you know. Otherwise, we could end up saying that the natural numbers are closed under subtraction because whenever you subtract two numbers you get some number or other as a result.
